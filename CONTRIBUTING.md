# Contributing Guidelines

Thank you for your interest in contributing to our Simple Interest Calculator project!

## How to Contribute

### Reporting Bugs
If you find a bug, please create an issue with:
- A clear, descriptive title
- Steps to reproduce the bug
- Expected behavior
- Actual behavior
- Any relevant screenshots or error messages

### Suggesting Enhancements
We welcome ideas for new features or improvements. When suggesting an enhancement:
- Explain the problem you're trying to solve
- Describe your proposed solution
- List any alternative solutions you've considered

### Code Contributions
1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes**
4. **Write or update tests** if applicable
5. **Ensure your code follows the project style**
6. **Commit your changes**: `git commit -m "Add your descriptive commit message"`
7. **Push to your branch**: `git push origin feature/your-feature-name`
8. **Create a Pull Request**

### Pull Request Guidelines
- Provide a clear description of what the PR accomplishes
- Reference any related issues
- Ensure all tests pass
- Update documentation if needed
- Keep PRs focused on a single change

## Development Setup
1. Clone the repository: `git clone https://github.com/[your-username]/github-final-project.git`
2. Navigate to the project directory
3. The script can be run directly: `./simple-interest.sh`

## Code Style
- Use meaningful variable names
- Add comments for complex logic
- Follow bash scripting best practices
- Test your changes before submitting

## Community
- Be respectful and inclusive
- Follow our Code of Conduct
- Help others when you can
- Welcome new contributors

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome!

Thank you for helping make this project better!
